# Lesson 07: Ambient Asset Inventory

## Goal

Learn how media assets are organized and documented before they are used in an
automation or render workflow.

## Concepts

- asset inventory
- file naming
- relative paths
- source versus output
- attribution/readiness notes

## Setup

Use these assets:

- `assets/mp3/beach.mp3`
- `assets/mp3/breeze.mp3`
- `assets/mp3/cafe.mp3`
- `assets/mp3/city.mp3`
- `assets/mp3/crickets.mp3`
- `assets/mp3/fire.mp3`
- `assets/mp3/forest.mp3`
- `assets/mp3/rain.mp3`

## Steps

1. List every MP3 asset.
2. Group the assets by mood or environment.
3. Check whether names are lowercase and easy to reference.
4. Decide which assets would work best for a focus session.
5. Write one limitation note for using MP3s in classroom projects.

## Challenge

Create a small asset manifest in your notes with columns for file, theme,
expected use, and caution.

## Expected Outcome

You can prepare media assets for a workflow without copying or renaming large
files unnecessarily.

## Cleanup / Reset

Do not create edited audio files in the repo. Use a local ignored workspace for
experiments.
