# Lesson 05: Turn-Based Game State

## Goal

Use classroom game scripts to understand state, turns, rules, and expected
results.

## Concepts

- game state
- turns
- valid moves
- win conditions
- reset behavior

## Setup

Use these assets:

- `assets/batch/TIC-TAC-TOE.bat`
- `assets/batch/POKEMON.bat`

Read the files first. Run only copied versions in a local workspace.

## Steps

1. Identify the variables that store player choices or game state.
2. Find where the script checks for invalid input.
3. Find at least one win or end condition.
4. Trace one happy path and one invalid path.
5. Write a simple test idea for each path.

## Challenge

Convert one game rule into a Given/When/Then statement.

## Expected Outcome

You can connect game logic to test cases and explain why state must be reset
between tests.

## Cleanup / Reset

Delete local copied scripts and notes from `workspace/` when done.
