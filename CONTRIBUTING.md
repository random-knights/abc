# Contributing

READMORE accepts public-facing documentation contributions for architecture concepts, source attributions, contribution-worthy specs, and product-facing guides.

## Flow

1. Open a focused pull request with the smallest useful documentation change.
2. Keep internal operations, <ENV_VAR>, local paths, deployment mechanics, private identities, and security posture out of public docs.
3. Link related public docs when useful, but do not update READLESS roadmap state from a public-docs PR.
4. PRs land through the qa-kitt review path before merge.

Docs that need internal coordination should stay in READLESS until the owner ratifies a public version.
