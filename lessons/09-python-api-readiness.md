# Lesson 09: Python API Readiness

## Goal

Use small Python samples to learn what must be checked before code calls an
external API.

## Concepts

- URL endpoints
- demo keys and placeholders
- dependency imports
- response parsing
- safe output

## Setup

Use these assets for review:

- `assets/python/iss-tracking.py`
- `assets/python/asteroid-tracking.py`

The samples use public/demo endpoints. Network calls should be optional and
reviewed before running.

## Steps

1. Identify imported packages.
2. Find every URL.
3. Mark whether each endpoint is public, demo, or source-required.
4. Identify what data is printed.
5. Write a safety note for running API examples in class.

## Challenge

Draft a "before you run" checklist covering dependencies, network access, API
limits, and expected output.

## Expected Outcome

You can review a small API script and explain what would need approval before
using it in a real workflow.

## Cleanup / Reset

Do not commit virtual environments, downloaded data, or API responses.
